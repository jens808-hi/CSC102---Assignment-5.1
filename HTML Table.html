Debugging is a method of identifying the problem (aka bugs), isolating it at the source, correcting the errors in the code, and then testing it. Debugging helps troubleshoot the root cause of coding errors, prevents software malfunctions and improves performance. Debugging is done when a developer finds coding errors in a program and is able to reproduce it. When attempting to identify bugs, an integrated development environment (IDE) is a very useful tool, highlighting lines of code corresponding to error messages simutaneously in real-time effectively pinpointing the root of the problem. Software Development refers to to a process where a series of tasks are dedicated to creating, designing, deploying, testing, maintaining, and supporting software applications. The function of these processes is to create programmable software to cater to the business or personal objectives and goals through writing computer code.

There are 6 stages to the Software Development Cycle (SDLC):
1. Planning
2. Analysis
3. Design
4. Implementation
5. Testing & Integration
6. Maintenance

Popular Software Development Processes are:
-Waterfall Development: Steps are executed linearly and sequentially in a waterfall approach.
-Agile Development: Focues on flexibility, continuously evolving functionality by an incremental approach.
-Rapid Application Development (RAD): Focuses prototyping on speed and flexibility over strict planning.
-Rational Unified Process (RUP): Framework that is tailored by organizations to best suit the specific needs of different projects.

Strings hold data that represent a sequence of characters, words or symbols as a single unit that are enlcosed in quotation marks. Strings can be stored as a constant or a variable that is fixed to an element or allows it be changed. String manipulation refers to the process of modifying, extracting, or transforming strings. One example is concatenation; meaning to join two or more strings together. Slicing strings to get only a portion of a string is an example of extracting strings. By changing specific parts of a string you can replace characters of a string. Converting a string to upper and lower case is an example of transforming strings. You can split a string by breaking it into an array of substrings. Using the trim feature you can remove leading or trailing whitespaces. Here are some string manipulation methods:

-String.charAt()
-String.lenght
-String.substr()
-String.replace()

Input Validation is the process of testing and validating user input, verifying if it meets a specific criteria and is in the correct format. A loop is iterated until the data is false or bad to determine the validation of user input. Empty input or wrong input are types of errors. Common errors are:

-state abbreviations (2 character strings only)
-zipcodes (5 character strings minimum, 9 characters maximum)
-wages (numerical values only)
-dates (numberical values usually)
-time (numerical values)

Data Integrity may be compromised if Input Validation is not ensured; resulting in data corruption, processing the wrong information, inaccuracy of outputs, unreliable system leading to unauthorized access leaving your system vulnerable to security attacks that may inject harmful code. Which in turn can tamper with the software and/or steal your data. 		Object Oriented Programming (OOP) is a model for writing programs and programming languages based on abstraction of objects, through this process a programmer conceals the data and code except for what is relevant to an object in order to reduce complexity, making it easier to work with. This can contain data in fields known as attributes or properties and code in procedures known as methods. For instance a button class has variables for data and functions, which can be inherited by a subclass and created as a subset of the button class. Objects are instances of classes. Using a concept known as structure or struct we can store multiple variables in one name:

    var size;
    var coat;
    var color;
    var name;
    var weight;
}
